Security

Authentication

Public Read — API Key Header

The opportunities dataset is publicly readable. All requests require the apikey header set to the platform's public anon key. No private credentials are needed for read access — the key carries read-only permissions enforced by Row Level Security.

Write operations (insert, update, delete) are blocked for the anon key. Contact us for a service-role key if you need write access.

Reference

Endpoints

GET /opportunities List all opportunities

Returns a paginated list of opportunities. Use PostgREST query parameters to filter by sector, status, type, or region. Use select= to specify which columns to return. Use Prefer: count=exact to get the total row count in the Content-Range response header.

Key Query Parameters
ParameterTypeRequiredDescription
select string optional Comma-separated column names. Defaults to * (all columns).
sectors=cs.{value} string optional Filter by sector (array contains). E.g. sectors=cs.{ict}
status=eq.value string optional open | closed_but_recurring | recurring
type=eq.value string optional grant | tender | accelerator | fellowship | investment_program
eligibility_region=eq.value string optional Saudi_only | GCC | Global | Global_including_Saudi
order string optional E.g. order=last_verified.desc
limit integer optional Results per page, max 1000 (default: 100)
offset integer optional Row offset for pagination (default: 0)
Example Request
curl "https://dshrbbnjahjcwxzvzygh.supabase.co/rest/v1/opportunities?select=id,title,type,status,sectors,eligibility_region&sectors=cs.%7Bict%7D&status=eq.open&limit=5" \
  -H "apikey: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Prefer: count=exact"

GET /opportunities?id=eq.{id} Get a single opportunity

Retrieve the full record for a single opportunity by its unique ID (e.g. SA-001). PostgREST returns an array — add Accept: application/vnd.pgrst.object+json to receive a single object instead.

GET /opportunities?sectors=cs.{ict} Filter by sector

Filter opportunities to a specific sector using PostgREST's array-contains operator cs.{value}. Combine with status=eq.open to surface only currently active programs. Available sectors: ict, energy, financial, innovation, healthcare, industrial, education, space, and more.

GET /opportunities?select=last_verified&order=last_verified.desc&limit=1 Dataset freshness

PostgREST exposes the raw table directly — use Prefer: count=exact to get the total row count and sort by last_verified to check the most recent pipeline run. There is no separate /stats endpoint — query the table directly.

Example — Total count + latest verification date
# Total record count (read Content-Range header)
curl "https://dshrbbnjahjcwxzvzygh.supabase.co/rest/v1/opportunities?select=id&limit=1" \
  -H "apikey: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Prefer: count=exact" \
  -I
# → Content-Range: 0-0/4116

# Most recent pipeline run
curl "https://dshrbbnjahjcwxzvzygh.supabase.co/rest/v1/opportunities?select=last_verified&order=last_verified.desc&limit=1" \
  -H "apikey: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
# → [{"last_verified":"2026-05-13"}]
